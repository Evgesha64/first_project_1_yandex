## Техническое задание к проекту М1

## 1. Общая информация и цель проекта
В этом проекте предполагается работа с датасетом из открытого источника. Нужно разработать модель машинного обучения, а также подготовить библиотеку и интерфейс к ней для предсказания на тестовой выборке.

В начале проекта доступны:
- обучающий датасет (train);
- тестовый датасет (test), **без правильных ответов**;
- скрипт, которым будет измеряться финальная метрика модели.

Метрика на тестовом датасете измеряется в ходе проверки работы.

## 2. Задачи проекта
В ходе работы над проектом должны быть решены следующие задачи:
- **Исследование датасета**: при необходимости — поиск дополнительной информации для лучшего понимания природы данных.
- **Предобработка данных**: возможны бесполезные признаки, сильная корреляция признаков, косвенные утечки целевого признака; некоторые колонки потребуется привести к корректным типам данных.
- **Обучение модели**.
- **Подготовка предсказаний на тестовой выборке**.
  - **Файл предсказаний**: CSV, 2 столбца — `id` и `prediction`.
- **Подготовка скриптов и библиотеки** для обработки данных и получения предсказаний на тестовой выборке.
- **Инструмент для тестирования**.
- **Документация**.

## 3. Описание исходных данных
Данные пациентов для предсказания риска сердечных приступов.

Содержимое (в общем виде):
- **id**: идентификатор записи.
- **Антропометрические параметры**: вес, возраст, рост.
- **Привычки**: курение, качество сна и т. д.
- **Давление**.
- **Наличие хронических заболеваний**.
- **Биохимия крови**.
- **Таргет**: высокий или низкий риск поражения сердца.

Доступные артефакты:
- `Train`
- `Test`
- скрипт, которым будет тестироваться качество модели

## 4. Метрика оценки качества и сравнения моделей
В ходе анализа датасетов нужно самостоятельно определить наилучшую метрику качества и обосновать выбор.

## 5. Стек
- Python (**ООП**)
- Pandas
- NumPy
- Scikit-learn
- CatBoost
- FastAPI
- HTML

## 6. Таймлайн проекта
Проект длится 2 недели и включает в себя:
- установочную встречу;
- семинар по FastAPI + QA-сессию;
- финальную встречу.

## 7. Результат и требования к оформлению
В результате должно получиться приложение на FastAPI (или аналогичном фреймворке), которое:
- принимает на вход путь к CSV-файлу тестовой выборки;
- выполняет предсказание (например, через POST-запрос к сервису);
- возвращает ответ в формате JSON.

Допускается для тестирования:
- написать скрипт, который посылает запрос к запущенному приложению;
- реализовать веб-интерфейс.

Дополнительно оценивается (помимо качества модели):
- структурированность, понятность и чистота кода;
- архитектура приложения;
- обязательное использование **ООП** (составные части приложения должны быть реализованы в виде классов).

Проект должен быть оформлен в виде репозитория на GitHub/GitLab и содержать документацию:
- для пользователя (как запустить и использовать);
- для разработчика (описание классов и методов приложения), как при передаче проекта дальше.

Важно:
- В Jupyter Notebook — только исследования и эксперименты.
- Если демонстрацию работы приложения вы выносите в ноутбук, **кода приложения в ноутбуке быть не должно**: он должен импортироваться как внешние модули.

## Формат сдачи работы
В репозитории должны быть:
- Jupyter Notebook с исследованием, обучением моделей и выводами;
- код приложения;
- предсказания на тестовой выборке в требуемом формате;
- инструкция по запуску приложения **или** Jupyter Notebook с демонстрацией работы приложения **или** демонстрация на финальной встрече.

## Критерии оценки
- исследование и результаты обучения моделей;
- метрика на тестовой выборке (измеряется при проверке);
- чистота и структурированность кода;
- оформление репозитория;
- воспроизводимость результатов.
